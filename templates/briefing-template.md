# Briefing template

Copy this structure into `briefings/YYYY-MM-DD.md` for each run. See `briefings/README.md` for the current focus project and the scope rule.

## Changelog digest

What's landed in the focus project's `CHANGELOG.md` entries since the last briefing.

## Specs digest

Status moves in the focus project's `specs/` (Draft → Active → Done).

## File freshness scan

Notebooks or specs in the focus project untouched for a while, flagged as worth a second look. Not auto-acted on — just surfaced.

## `check_guide()` reminder

A prompt to run it yourself in a notebook and paste the output back. Not run automatically.

## Literature scan

3-5 papers on the focus project's theme. Aim for 1-3 recently posted plus a couple from the last 5-10 years not yet reviewed. Checked against `docs/papers/` (the shared reference library) to avoid repeats. Free to pull in adjacent VDFI subprojects' topics too, since the papers folder isn't scoped to one project.

**Citation format — always lead with `Lastname Year`**, matching the `docs/papers/` filename convention, so each paper is instantly recognizable by name rather than by title:

> **Lastname Year** — Full title (arXiv:XXXXX, Month Year). 2-3 sentence summary.

Example:

> **Trainor 2025** — The Lyman-alpha Halos of Galaxies at z=2-3 in the Keck Baryonic Structure Survey (arXiv:2505.15881, May 2025). ...
