# Briefings

A weekly digest, saved as one dated markdown file per run (e.g. `2026-07-13.md`). Lives at the workspace root — not inside any one subproject — because the literature-scanning section deliberately looks wider than a single subproject, even though the rest of the briefing stays focused on one.

This is a solo-use log, not a team status update: it exists so Austin has a running, git-trackable record to scroll back through, not to surface changes to anyone else.

## Current focus project

`01-lya_halos`

Update this line whenever the focus shifts to a different subproject. The briefing reads this file each run to know what to scope the changelog/specs/freshness sections to — no need to re-specify it each time.

## Format

See `templates/briefing-template.md` for the section-by-section structure every briefing follows, including the citation format for the literature scan.

## Scope rule

Changelog / specs / freshness = focus project only. Literature = focus project first, adjacent topics welcome, checked against the workspace-wide `docs/papers/` library either way.
